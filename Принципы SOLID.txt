1. Принцип единой ответственности. 
   Класс GuessNumber отвечает только за логику игры: генерация числа, проверка числа, подсказки. Класс Settings отвечает только за хранение настроек. Класс ConsoleInput отвечает только за вывод сообщений. Класс ConsoleOutput отвечает только за ввод пользователем данных.

2. Принцип инверсии зависимостей.
   Класс GuessNumber зависит только от интерфейсов и не зависит от их реализации, что позволяет менять генератор чисел, способ вывода подсказок и способ ввода данных пользователем.

3. Принцип разделения интерфейса.
   Интерфейс INumberGenerator определяет только метод генерации чисел. Интерфейс IInput определяет только метод вывода сообщений. Интерфейс IOutput определяет только функцию ввода данных. В результате класс GuessNumber зависит только от необходимых ему функция.

4. Принцип открытости/закрытости.
   Класс GuessNumber открыт для расширения различными методами генерции чисел, ввода, вывода, но закрыт от изменения логики игры.	

5. Принцип подстановки Барбары Лисков.
   Класс GuessNumber работает с интерфейсами, что позволяет подставить любой класс, наследующий этим интерфейсам.
